# Contributing

## How to Contribute

1. Fork the repository
2. Create a feature branch from `main`
3. Make your changes
4. Ensure all quality checks pass
5. Submit a pull request

## Guidelines

### Code Changes

- **Generalize, don't replace**: Our code may have hardcoded values for our specific use cases. Don't just replace them with your own values - generalize the code to support both.
- **Add tests**: All new functionality must have associated tests
- **Pass all checks**: Your code must pass all quality checks before submission

### Quality Requirements

Your code must pass all quality checks. See CODE_QUALITY.md for detailed standards.

### AI-Assisted Code

AI-generated code is welcome provided:
- It includes comprehensive tests
- It passes all quality checks
- The PR clearly indicates AI assistance was used

### Commit Messages

- Use clear, descriptive commit messages
- Reference issue numbers where applicable
- Keep commits focused and atomic

### Pull Request Process

1. Update documentation if needed
2. Add tests for new functionality
3. Ensure CI passes
4. Request review from maintainers
5. Address review feedback promptly

## Code of Conduct

- Be respectful and professional
- Focus on technical merit
- Provide constructive feedback
- Welcome newcomers

## Questions?

Open an issue for questions about contributing.
